1. Resumen del Sistema

RECALDE es una aplicacion web MVC en PHP para gestion comercial y operativa de: clientes, inventario, pedidos, ventas, abonos, historial, perfiles y panel tecnico de desarrollador.

  • Backend: PHP 8.1+ sin framework pesado (MVC propio).
  • Base de datos: MySQL/MariaDB.
  • Frontend: HTML/CSS/JS vanilla + Chart.js.
  • Autenticacion: sesion PHP, control de roles, CSRF en rutas autenticadas POST.
  • Recuperacion de contrasena por correo con token temporal.

5. Flujo de Funcionamiento

5.1 Flujo de autenticacion

Usuario -> ?route=login -> validar-login (POST)
      -> LoginController::authenticate()
      -> Usuarios::getUserByUsername()
      -> password_verify()
      -> session_regenerate_id(true)
      -> $_SESSION['usuario']
      -> redirect dashboard/perfil

5.2 Flujo recuperar contrasena

forgot-password (GET)
  -> forgot-password-send (POST)
  -> Usuarios::createPasswordResetToken()
  -> Mailer::send() [smtp/mail/log]
  -> reset-password?token=...
  -> reset-password-save (POST)
  -> Usuarios::resetPasswordWithToken()

5.3 Flujo comercial (pedido a venta)

Cliente -> Pedido -> Detalle pedido (+medidas)
       -> Venta (desde pedido)
       -> Abonos
       -> Estado pago (pagado/parcial/pendiente)
       -> Historial de ventas

7. Rutas Principales

Todas las rutas pasan por index.php usando ?route=....

Publicas: login, validar-login, forgot-password, forgot-password-send, reset-password, reset-password-save.
Privadas (ejemplos): home, perfil, clientes, pedidos, ventas, inventario, historial, developer, developer-action, logout.

Nota: en rutas privadas con metodo POST se valida CSRF global.

8. Instalacion Local (Desarrollo)

  1. Copiar .env.example a .env.
  2. Crear base de datos RECALDE.
  3. Importar storage/schema.sql y storage/seed.sql.
  4. Configurar credenciales DB y correo en .env.
  5. Levantar servidor local: php -S 127.0.0.1:8000.
  6. Abrir: http://127.0.0.1:8000/?route=login.

8.1 Requisitos

  • PHP 8.1+ con pdo, pdo_mysql.
  • MySQL 8+ o MariaDB compatible.
  • Apache con mod_rewrite (en produccion recomendada).

9. Despliegue en Hostinger (Shared Hosting)

9.1 Paso a paso recomendado

  1. En hPanel crea la base de datos MySQL y usuario DB.
  2. Importa storage/schema.sql y storage/seed.sql por phpMyAdmin.
  3. Sube el proyecto por File Manager o FTP.
  4. Configura .env de produccion (APP_DEBUG=0, APP_URL=https://tu-dominio.com).
  5. Configura correo SMTP real para recuperacion de contrasena.
  6. Verifica permisos de storage/ para escritura del servidor web.
  7. Prueba login, CRUD basicos y recuperacion de contrasena.

9.2 Opcion A (ideal): DocumentRoot al directorio public/

  • Si tu plan permite custom document root, apunta a /ruta/recalde/public.
  • Esta es la forma mas limpia y segura para produccion.

9.3 Opcion B (si no puedes cambiar DocumentRoot)

  • Publicar en public_html completo.
  • El proyecto ya incluye reglas .htaccess para bloquear rutas sensibles.
  • Aun asi, valida manualmente que no se expongan .env, storage/, src/, config/, vendor/.
En hosting compartido, nunca subas .env con credenciales de desarrollo. Usa credenciales nuevas de produccion.

9.4 Variables .env minimas para hosting

APP_NAME=RECALDE
APP_VERSION=1.2.2
APP_DEBUG=0
APP_URL=https://tu-dominio.com

DB_HOST=localhost
DB_NAME=TU_DB
DB_USER=TU_USER
DB_PASS=TU_PASSWORD
DB_CHARSET=utf8mb4

MAIL_DRIVER=smtp
MAIL_FROM_ADDRESS=no-reply@tu-dominio.com
MAIL_FROM_NAME=RECALDE
MAIL_HOST=smtp.tu-proveedor.com
MAIL_PORT=587
MAIL_USERNAME=usuario_smtp
MAIL_PASSWORD=clave_smtp
MAIL_ENCRYPTION=tls
MAIL_LOG_PATH=/tmp/recalde-mail.log

10. Configuracion de Correo (Recuperacion de Contrasena)

  1. Configura SMTP real en .env.
  2. Verifica que el puerto SMTP este permitido en tu plan.
  3. Si usas Outlook/Gmail, usa clave de aplicacion cuando aplique.
  4. Prueba ?route=forgot-password con un correo existente.
Si el envio falla, el sistema deja un log redactado en MAIL_LOG_PATH (sin exponer el token completo de recuperacion).

12. Soporte y Mantenimiento

12.1 Archivos clave para soporte tecnico

  • index.php: enrutamiento y seguridad global.
  • src/controllers/dashboardcontroller.php: flujo principal del dashboard.
  • src/controllers/logincontroller.php: login y recuperacion.
  • src/models/usuarios.php: usuarios, hash y reset token.
  • src/core/mailer.php: transporte de correo.
  • storage/schema.sql: estructura de BD.

12.2 Recomendacion de versionado

  • No versionar .env ni logs.
  • Usar ramas: develop y main.
  • Hacer backup antes de aplicar cambios en produccion.
